ES-6 Modules,no Router




Reflections

-The only element changing when pressing the buttons is the "content" div. This is defined in the handleButtonClicks function
-The concept of ES6 modules is to split up javascript files to give better oversight. Creating 1 javascript file for 5 pages is worse than creating 5 js files for 5 pages.
With 5 different js files, you know which functions and usage the javascript inside have. And you dont lose any functionality, because all functions can be exported/imported
-HTML is loaded when page is loaded, so all the elements are there to be "edited" through the DOM. The javascript is loaded using ES6 modules and the function is only loaded when it is called in our onclick handleButtonClicks function. -The term Singleton is used for javascript modules because you only a single instance of that module exists. It can then be imported and used globally in the entire application.
